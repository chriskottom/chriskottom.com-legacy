If you want to use even the most simple tool effectively, you need to know how to hold it. That's why I'm writing The Minitest Cookbook - to help Ruby and Rails developers at all skill levels master the techniques they need to write clear, maintainable tests and help others to do the same. That process begins with a set of fundamentals where you'll learn how to:

  • Write tests using both assertions and specs
  • Structure your tests for clarity and confidence
  • Implement boundary checks, fuzzing, and other testing techniques
  • Validate your code with assertions and expectations
  • Use mocks, stubs, and other kinds of doubles
  • Effectively manage test data
  • Share common code between tests
  • Run one test case or your whole suite

Once you've covered the basics and your feel you're ready to move to the next level, we've got you covered, and so does Minitest. Because even though it's compact, Minitest is also super flexible and ready to test whatever sort of Ruby code you want to throw at it. It's based around a simple architecture and supports customizations via plugins, so you'll be able to trick out your testing stack with third-party gems from a large and growing ecosystem of plugins. Later sections of the book will deal with some more advanced topics on a deeper level including:

Rails application testing

Piecing together the right way to test your Rails projects relying only on a bunch of disconnected blog posts and READMEs is a sure way to make your head hurt. Let me show you how to set up your testing stack and practical recipes for testing every part of your application.

Writing and sharing your own extensions

Minitest's compactness and simplicity offer plenty of opportunities for customizing and changing the framework's basic functions. We'll peek into the source code and look at some popular plugins to see how you can extend Minitest to meet your own needs.

Optimizing test performance

Developers will run a fast test suite more often than a slow one, and tests that run more often are more effective than those that don't. I'll show you how to speed up your existing tests as well as the techniques and tricks to avoid writing slow tests in the first place.

Building your own testing DSL

Minitest's standard assertions and expectations are a great start, but you see that you can build up a highly targeted custom testing vocabulary that maps closely to your application's domain and allows you to write tests that are more expressive and compact.
